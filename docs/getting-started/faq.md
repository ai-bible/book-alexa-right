# Часто задаваемые вопросы (FAQ)

## Общие вопросы

### Что такое AI Writing System?

AI Writing System — это комплексная система для автоматизированного создания литературного текста научно-фантастического романа с использованием специализированных ИИ-агентов и Model Context Protocol (MCP).

### Для кого эта система?

Система предназначена для:

- Авторов, пишущих объёмные произведения (романы, сериалы)
- Тех, кто хочет систематизировать процесс планирования и написания
- Писателей, работающих с complex worldbuilding и multiple storylines
- Разработчиков, интересующихся AI-assisted creative writing

### Нужно ли мне знать программирование?

**Для использования** — не обязательно. Система работает через простые команды в Claude Code.

**Для расширения** — базовые знания Python помогут создавать собственных агентов и hooks.

### Сколько стоит использование?

Сама система open-source и бесплатна. Вам потребуется:

- Доступ к Claude Code (см. документацию Anthropic)
- Обычный компьютер (минимум 4GB RAM)

## Планирование

### Как долго длится планирование сцены?

Обычно **2-5 минут** на одну сцену, включая:

- Анализ контекста (30 сек)
- Генерация сценариев (1 мин)
- Детализация (1-2 мин)
- Интеграция (30 сек)

### Могу ли я пропустить этап планирования?

Нет, blueprint обязателен для генерации. Но вы можете:

- Создать минимальный blueprint вручную
- Использовать существующий blueprint как template
- Запустить быстрое планирование с меньшей детализацией

### Что делать если сгенерированные сценарии не нравятся?

1. Попросите сгенерировать новые варианты
2. Уточните свои цели и constraints
3. Отредактируйте blueprint вручную

### Можно ли изменить blueprint после создания?

Да, но помните:

- Изменения могут инвалидировать уже сгенерированный текст
- Система покажет warning о несогласованности
- Придётся пересмотреть зависимые blueprints

## Генерация

### Сколько времени занимает генерация сцены?

**5-8 минут** на сцену (2,000-3,000 слов), включая:

- Validation (1 мин)
- Human approval (30 сек)
- Generation (2-4 мин)
- Quality checks (2 мин)

### Что если сгенерированный текст низкого качества?

**Немедленные действия**:

1. Проверьте blueprint — достаточно ли деталей?
2. Проверьте validation report — какие проблемы обнаружены?
3. Повторите генерацию (система автоматически улучшит подход)

**Долгосрочные улучшения**:

1. Улучшите детализацию blueprints
2. Обновите character cards с более детальными voice patterns
3. Добавьте примеры желаемого стиля в prose style guide

### Можно ли изменить стиль написания?

Да, через [`prose-style-guide.md`](../technical/prose/style-guide.md):

- Tone и voice
- Sentence structure preferences
- Vocabulary level
- Литературные приёмы

### Сколько попыток генерации допустимо?

Система автоматически делает до **3 попыток** при сбоях.

Вы можете повторять генерацию вручную неограниченно, пока не получите нужный результат.

### Что делать если генерация упала?

Используйте recovery механизм:

```bash
# Проверить статус
/generation-state status scene-0204

# Восстановить с последнего шага
/generation-state resume scene-0204
```

Система автоматически продолжит с места падения.

## Сессии и версионирование

### Зачем нужны сессии?

Сессии позволяют:

- Экспериментировать с альтернативными сюжетами
- Откатываться к предыдущим версиям
- Работать параллельно над разными вариантами
- Безопасно тестировать идеи

### Как создать новую сессию?

```bash
/session-create experiment-ending
/session-activate experiment-ending
```

Теперь все изменения будут в отдельной ветке.

### Можно ли объединить сессии?

Да, через merge:

```bash
/session-merge experiment-ending main
```

Система покажет conflicts, которые нужно разрешить вручную.

### Сколько сессий можно создать?

Неограниченно, но помните о дисковом пространстве — каждая сессия создаёт копию файлов при изменении (Copy-on-Write).

## Контекст и консистентность

### Как система отслеживает консистентность?

Через **5 уровней канона** (0-4):

- Level 0: Immutable core (нельзя менять)
- Level 1: Fundamental lore (очень осторожно)
- Level 2: Established canon (можно, но с проверками)
- Level 3: Working details (свободно)
- Level 4: Drafts (экспериментально)

### Что делать при обнаружении противоречия?

Система покажет **consistency warning**:

1. Проверьте какой элемент важнее
2. Решите что изменить
3. Обновите соответствующие файлы
4. Запустите `/check-consistency` для проверки

### Как обновлять knowledge персонажей?

Автоматически через Integration Workflow после генерации.

Вручную:

1. Отредактируйте `context/characters/{name}.md`
2. Добавьте новые события в timeline
3. Обновите knowledge состояние

### Можно ли изменить важные элементы мира?

Да, но учитывайте canon level:

- **Level 0-1**: Требуют пересмотра всего романа
- **Level 2**: Требуют проверки зависимостей
- **Level 3-4**: Свободно

## Технические вопросы

### Какие системные требования?

**Минимальные**:

- Python 3.13+
- 4GB RAM
- 10GB дискового пространства
- Интернет (для Claude API)

**Рекомендуемые**:

- 8GB+ RAM
- SSD
- 50GB+ свободного места

### Почему генерация медленная?

Возможные причины:

1. **Сетевая задержка** — проверьте интернет-соединение
2. **Сложный blueprint** — упростите детализацию
3. **Многочисленные персонажи** — рассмотрите разбивку на сцены
4. **Большой контекст** — используйте `/compact` для оптимизации

### Где хранятся данные?

```
book-alexa-right/
├── acts/           # Сгенерированный текст
├── context/        # Мир, персонажи, локации
├── workspace/      # Временные файлы и логи
└── mcp-servers/    # Состояние workflows (SQLite)
```

**Важно**: Регулярно делайте backup через Git.

### Можно ли использовать свою модель?

Система построена на Claude Code, который использует Claude API.

Для других моделей потребуется:

1. Переписать MCP серверы
2. Адаптировать prompt templates
3. Настроить validation logic

### Как просмотреть логи?

Логи хранятся в `workspace/logs/`:

```bash
# Последние ошибки
tail -f workspace/logs/generation/errors.log

# Specific agent logs
cat workspace/logs/prose-writer/session-{timestamp}.log
```

### Поддерживается ли Windows?

Да, через WSL2 (Windows Subsystem for Linux).

Native Windows support — частично (могут быть проблемы с путями).

## Troubleshooting

### Ошибка: "Blueprint not found"

**Причина**: Файл blueprint отсутствует.

**Решение**: Запустите `/plan-story` для создания blueprint.

### Ошибка: "Parent not approved"

**Причина**: Родительский план (Chapter/Act) не утверждён.

**Решение**: Сначала утвердите родительский план через hierarchy validation.

### Ошибка: "Validation failed"

**Причина**: Blueprint содержит errors.

**Решение**: Откройте validation report, исправьте указанные проблемы.

### Ошибка: "Context overflow"

**Причина**: Слишком много данных в контексте.

**Решение**:

```bash
/compact        # Сжать контекст
/clear          # Очистить историю (осторожно!)
```

### Генерация зависла

**Решение**:

1. Подождите 5 минут (может быть долгая операция)
2. Проверьте логи: `workspace/logs/generation/`
3. Отмените и перезапустите:

```bash
/generation-state cancel scene-0204
# Затем повторите
Сгенерируй сцену 0204
```
